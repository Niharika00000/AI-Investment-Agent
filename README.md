# InvestInsight — AI Investment Research Agent

An autonomous, multi-agent AI system that conducts deep equity research on any company and delivers a clear **INVEST ✅ or PASS ❌** verdict with full reasoning.

Built with **Next.js 15**, **Node.js/Express**, **LangGraph.js**, **Google Gemini / OpenAI**, **PostgreSQL (Prisma)** — the exact production stack requested by InsideIIM × Altuni AI Labs.

---

## Overview

InvestInsight takes a company name as input, then autonomously:

1. **Researches the company** — profile, leadership, business model, SEC filings
2. **Analyzes financials** — revenue growth, margins, FCF, P/E, debt ratios
3. **Reads market sentiment** — news articles, analyst ratings, sentiment scoring
4. **Benchmarks competitors** — moat, market share, peer comparison
5. **Evaluates risks** — regulatory, market, macro, and company-specific
6. **Issues a verdict** — weighted composite decision with confidence score

The result is rendered in a premium, fully responsive dashboard with interactive charts, a dark/light theme switcher, and an explainable AI breakdown.

## System Architecture

### Multi-Agent Pipeline (LangGraph.js)

```
START → Company Research → Financial Analysis → News Sentiment
      → Competitive Analysis → Risk Analysis → Investment Decision → END
```

Each agent is a dedicated node in a **LangGraph StateGraph**. State flows forward — every agent reads prior results and adds its own output. The final Investment Decision agent synthesizes everything into a single recommendation.

## How to Run

### Prerequisites
- **Node.js 20+**
- **PostgreSQL 14+** running locally (create a database named `investdb`)
- **Google Gemini API Key** — free at [Google AI Studio](https://aistudio.google.com)

### 1. Backend Setup

```bash
cd server
cp .env.example .env
# Edit .env with your credentials (see configuration below)
npm install
npm run prisma:push
npm run prisma:generate
npm run dev
```

Backend runs on `http://localhost:3001`

### 2. Frontend Setup

```bash
cd client
npm install
# Verify client/.env.local has: NEXT_PUBLIC_API_URL=http://localhost:3001
npm run dev
```

Frontend runs on `http://localhost:3000`

### 3. Create a test account

Visit `http://localhost:3000/signup` and register. Then use those credentials to log in at `/login`.

## How It Works — Architecture Deep Dive

### Agent Pipeline

Each of the 6 agents follows the same pattern:

```
Fetch structured data from APIs
  → Build a context prompt
  → Invoke LLM with Zod-validated structured output schema
  → Return partial GraphState update
```

The `LangGraph StateGraph` manages channel-level state merging, so agents run sequentially with access to all prior outputs.

### Structured LLM Outputs

Every agent uses `llm.withStructuredOutput(zodSchema)` which forces the LLM to return valid, typed JSON that Zod validates. This eliminates hallucination risk on structured fields (scores, enums, arrays).

### Decision Logic — Weighted Composite Score

```
Composite = (Financial × 0.30) + (CompanyQuality × 0.20)
          + (Competitive × 0.20) + (Sentiment × 0.15) + ((100−Risk) × 0.15)

INVEST ✅  →  Composite ≥ 60
PASS   ❌  →  Composite < 60
```

Each sub-score is generated by the respective specialist agent and is independently explained.

### Multi-Provider LLM Support

The `createLLM()` factory in `server/src/lib/llm.ts` supports hot-swapping between Google Gemini, OpenAI, and Anthropic via `LLM_PROVIDER` in `.env`. No code changes required.

## Key Decisions & Trade-offs

| Decision | Rationale | Trade-off |
|---|---|---|
| **LangGraph for orchestration** | Explicit state transitions, debuggable agent flow, production-grade | More setup than simple function chaining |
| **Sequential agent pipeline** | Each agent enriches the shared state for the next | Longer total runtime (~30-60s with real APIs); parallel would be faster but agents need prior outputs |
| **Zod for structured outputs** | Type-safe LLM responses, eliminates JSON parse errors | Schema verbosity; some LLM providers (Gemini) have payload serialization quirks with nested arrays |
| **PostgreSQL + Prisma** | Full research history per user, efficient indexed queries | Requires local DB setup; SQLite could be used for zero-config |
| **Mock Mode fallback** | Enables instant UI testing / demo without API keys | Mock data is static; not useful for real research |
| **JWT auth (stateless)** | Simple, scalable, no session store needed | Token expiry requires re-login; no refresh token flow |

**What was left out:**
- WebSocket / SSE streaming (research currently runs synchronously; adding SSE would let the UI show real-time agent progress)
- Redis caching (currently uses in-memory Map cache; Redis would survive restarts and scale horizontally)
- Vercel deployment (requires cloud PostgreSQL; can be added with PlanetScale/Supabase)
- Rate limiting per user (currently limits by IP)
